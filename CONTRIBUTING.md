# Contribuindo para o Off3D Studio Management UI 🚀

Primeiramente, muito obrigada por demonstrar interesse em colaborar! Este projeto nasceu para ser um espaço seguro de aprendizado e protagonismo, especialmente para **mulheres na tecnologia** e pessoas aliadas. 💜

Aqui, acreditamos que nenhum código é pequeno demais e que a colaboração mútua é o melhor caminho para o crescimento profissional.

---

## 🛠️ Como começar?

### 1. Preparando o ambiente
Certifique-se de ter instalado:
* **Node.js** (Versão 18 ou superior)
* **Angular CLI** (`npm install -g @angular/cli`)
* **Git**

### 2. Configurando o projeto
1. Faça um **Fork** deste repositório.
2. Clone o seu fork: `git clone https://github.com/SEU_USUARIO/off3d-management-ui.git`
3. Entre na pasta: `cd off3d-management-ui`
4. Instale as dependências: `npm install`
5. Rode o projeto: `ng serve` (Acesse em `http://localhost:4200`)

---

## 🌳 Padrão de Branches e Commits

Para mantermos a organização, utilizamos o padrão de **Conventional Commits**:

* `feat:` para novas funcionalidades (ex: `feat: add login screen`)
* `fix:` para correção de bugs (ex: `fix: resolve auth routing`)
* `docs:` para mudanças na documentação
* `style:` para formatação e estilo de código (CSS/SASS)

**Nome das Branches:**
* Se for uma funcionalidade: `feature/nome-da-task`
* Se for um bug: `fix/nome-do-bug`

---

## 🚀 Ciclo de Contribuição

1. **Escolha uma Issue:** Verifique as abas de "Issues" do repositório ou crie uma nova se encontrar algo para melhorar.
2. **Crie sua Branch:** `git checkout -b feature/minha-nova-tela`
3. **Desenvolva e Teste:** Garanta que o build está passando com `npm run build`.
4. **Push e Pull Request:** Faça o push para o seu fork e abra um **Pull Request (PR)** para a branch `develop` do repositório original.

> **Importante:** Descreva o que você fez no PR! Se puder, anexe um print da tela que você construiu.

## 💜 Código de Conduta e Cultura

* **Espaço Seguro:** Não tenha medo de errar ou de fazer perguntas "básicas". Estamos todas aprendendo.
* **Review Construtivo:** Ao revisar o código de uma colega, seja gentil e técnica. O foco é a evolução do código e da dev.
* **Clean Code:** Como o backend segue padrões rigorosos (SonarQube), buscamos manter o frontend organizado, legível e modularizado.
